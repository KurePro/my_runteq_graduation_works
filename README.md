# [Fridge Note] - 冷蔵庫在庫管理アプリ

## サービス概要
一人暮らしの方向けに、冷蔵庫内の食材を「直感的に」「数秒で」管理できるWebアプリケーションです。
既存の在庫管理アプリの「登録が面倒」「更新がだるい」という課題を、モダンなフロントエンド技術（Hotwire）を用いた非同期処理とUXの工夫によって解決しました。

#### **【ゲストログイン】**
初期データがセットアップされるゲストログインがあります。
- URL: [Fridge Note](https://my-runteq-graduation-works.onrender.com)

---

## 制作背景
冷蔵庫の奥から1年前に賞味期限が切れた「鮭のふりかけのビン」を発見したことがきっかけです。
日々の仕事に追われる若年層にとって、食材の期限管理は「面倒な作業」になりがちです。そこで、「買い物のついでに、自然と冷蔵庫の在庫が更新される」仕組みがあれば、食品ロスを減らし、日々の買い出しも効率化できると考え開発に至りました。

---

## コア機能とこだわったポイント！

### 1. 買い物リストから冷蔵庫への「ゼロ遷移」移動
買い物リストで「買った」チェックを入れると、画面遷移なしでモーダルウィンドウが立ち上がります。
UX向上のため、**Hotwire (Turbo Frames / Streams) と Stimulus、そしてHTMLの`<dialog>`要素を連携**させ、ストレスフリーなUIを実装しました。
また、トランザクション処理を用い、リストからの削除と食材一覧への登録を安全に行っています。

### 2. 直感的なステータス管理と柔軟な単位
単なる「個数」だけでなく、肉や調味料など「使った分だけ減らす」ことができるよう、gやmlなどの単位に対応。
在庫状況は `enum` を用いて「在庫あり / 残りわずか / 在庫切れ / 期限切れ」と明確にステータス化し、色分け表示で一目で把握できるように工夫しています。

# 下記は、アプリ制作時の過去のREADMEです。
# my_runteq_graduation_work
2025.11.1~ RUNTEQ卒業制作／冷蔵庫在庫管理アプリ

## サービス概要

一人暮らし向けに、冷蔵庫内の食材をシンプルに登録・管理できるアプリ。

「登録が面倒」「更新がだるい」「単位が個数しかない」といった既存アプリの不便を解消し、
**毎日数秒で在庫を更新できる**ことを目指す。


## このサービスへの思い・作りたい理由

冷蔵庫の掃除をしていた際、1年前に期限が切れた鮭のふりかけのビンが見つかった。
その他にも、期限切れとはいえないまでも、忘れかけていた品があり、一括で管理できる上に通知が来るサービスに思い至った。


## ユーザー層について

仕事に追われる若年層。
直感的な操作で、登録が苦にならない、続けやすいUIを心がける。

将来的には家族層まで。
初期実装は一人暮らしターゲットの機能までにして、最終的には家族共有等も実装したい。


## サービスの利用イメージ

食材を余らせないようになる。また、冷蔵庫の中身を把握出来るため、買い物もしやすくなる。


## ユーザーの獲得について

SNS。また、サイトのタイトルはわかりやすく、端的にサービスについて伝えられるものにする。


## サービスの差別化ポイント・推しポイント

他のサービスを確認すると、個数設定のみ対応のシステムであった。さらに、登録した食べ物をタップして数量を0にするという操作が必要で、若干の面倒さがある。

→肉や調味料等を、使った分登録し、残数がわかるようにする。gやmlに対応する。

→また、enumでステータスを「在庫あり、残りわずか、在庫切れ、期限切れ」等設定し、単純な登録も可能にする。
また、在庫が切れた食品を、すぐに買い物リストに追加できる機能をつける。

### テーブル詳細
#### Usersテーブル（例：ユーザー情報）
- name : ユーザーの表示名（山田太郎）
- email : ログイン認証用のメールアドレス / ユニーク制約 （taro@example.com）
- encrypted_password : ログイン認証用のパスワード
- avator ： ユーザーの任意の画像

#### Foodsテーブル（例：食品情報）
- name : 食品名（レタス）
- expiry_date : 消費期限 （2025/11/26）
- quantity : 食品の量
- unit ： 食品の数詞（個、g）
- status ： 在庫、期限の状況(enum/stored:0, expiring:1, expired:2, consumed:3)
- memo ： 備考（適当なメモ）

#### Categoriesテーブル（例：カテゴリー情報）
- name : カテゴリー名（肉類）

#### Notificationsテーブル（例：通知情報）
- title : 通知の題名（「〇〇の賞味期限が3日前です」）
- checked : 既読か
- notification_timing ： 通知タイミング管理(enum/expiry_3days:0, expiry_today: 1, expired: 2)

#### ShoppingItemsテーブル（例：買い物リスト情報）
- name : 買う予定の食品名（レタス）
- is_bought : 買ったフラグ（boolean）
- memo ： 備考
